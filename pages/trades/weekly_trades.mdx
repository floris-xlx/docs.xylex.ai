<Card title="Endpoint Description" icon="info" horizontal >Retrieves weekly trades for a specified organization and user, verifying user organization alignment. Returns trades that are not `unapproved` or `invalid` for the requested or current week. Handles errors and mismatches with appropriate HTTP responses.</Card>





<ParamField query="user_id" type="String" required  children="The unique identifier for the user making the request.">
  User identifier
</ParamField>
<ParamField query="organization" type="String" required  children="The name or identifier of the organization associated with the user.">
  Organization identifier
</ParamField>
<ParamField query="week_nr" type="Option<i64>"  children="An optional week number to filter the trades; defaults to the current week if not provided.">
  Week number
</ParamField>
<ParamField query="algorithm_id" type="String" required  children="The unique identifier for the algorithm used in the trade request.">
  Algorithm identifier
</ParamField>
