<Card title="Endpoint Description" icon="info" horizontal >Fetches weekly trades based on user ID, organization, and algorithm ID. Verifies user organization before retrieving trades from a database, filtering out unapproved or invalid trades. Responds with trade data or error messages for mismatches or data retrieval issues.</Card>





<ParamField query="user_id" type="String" required  children="A string representing the unique identifier of the user making the request.">
  User's unique identifier
</ParamField>
<ParamField query="organization" type="String" required  children="A string representing the name of the organization associated with the user.">
  Organization name
</ParamField>
<ParamField query="week_nr" type="Option<i64>"  children="An optional 64-bit integer representing the week number for which trades are fetched. If not provided, the current week is used.">
  Week number
</ParamField>
<ParamField query="algorithm_id" type="String" required  children="A string representing the unique identifier of the algorithm used for trades.">
  Algorithm ID
</ParamField>
